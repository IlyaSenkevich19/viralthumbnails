# Paywall and Upgrade A/B Experiment Ideas

Use this as a menu of testable hypotheses. Prioritize by expected lift, implementation cost, and statistical power (enough traffic per variant).

## Trigger and Timing

- Delay first paywall N sessions vs N value events (e.g., after first export, after second project).
- Hard gate on click vs soft preview with dismiss on first encounter.
- Trial warning cadence: single 3-day vs 7/3/1 multi-touch.
- Limit hit: block action vs allow with watermark vs allow with save disabled.

## Copy and Framing

- Gain frame ("Unlock unlimited exports") vs loss frame ("Don't lose your work") — test ethically; avoid manipulative loss copy.
- Feature-led headline vs outcome-led headline vs social-proof-led headline.
- Short body vs long body with one concrete example.
- "Pro" vs plan name tied to outcome (e.g., "Creator").

## Price Presentation

- Monthly default vs annual default with monthly equivalent.
- Order of plans: cheap-to-expensive vs recommended-in-the-middle.
- Show per-seat vs show total for team.
- Anchor: show higher tier first vs lead with most popular only.

## Trial and Offers

- Trial length: 7 vs 14 days (or product-appropriate).
- Credit card required vs not at trial start (if policy allows).
- Limited-time discount on first upgrade vs none.
- Add-on bundle (e.g., seats + storage) vs single plan upsell.

## Layout and Components

- Comparison table vs card stack vs minimal two-option chooser.
- Video or interactive preview vs static screenshot.
- Testimonial above fold vs below pricing.
- Single primary CTA vs secondary "Talk to sales" for enterprise.

## Upsell and Tier Motion

- Mid-session feature upsell vs settings-only upgrade path.
- Post-success moment (after publish, after share) paywall vs generic entry.

## Metrics and Guardrails

- Primary: upgrade conversion from paywall impression, or revenue per active user.
- Guardrails: dismiss rate, support tickets, refund/churn in first 30 days, session length, return rate.

## Analysis Notes

- Run long enough to capture weekly seasonality if traffic is low.
- Segment by platform, geography, and acquisition channel where sample size allows.
- Prefer pre-registered hypotheses and avoid peeking without correction.
