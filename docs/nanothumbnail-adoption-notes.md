# Идеи из NanoThumbnail для ViralThumbnails

Референс: [yoanbernabeu/NanoThumbnail](https://github.com/yoanbernabeu/NanoThumbnail) (MIT).  
Цель документа — зафиксировать **что имеет смысл перенять по смыслу** в наш Nest/Next стек, **без** копирования их клиентской архитектуры (BYOK в браузере, CORS proxy).

---

## Перенести (паттерны и фичи)

1. **Шаблон промпта** — фиксированная «оболочка» под YouTube (контраст, читаемость, стиль) + пользовательское описание + отдельная фраза под **текст на превью** (аналог их `bold text overlay saying '…'`).

2. **Разделение полей в UI/API** — не один общий textarea: минимум **креатив/контекст**, **заголовок на картинке**, при необходимости **стиль**; итоговый промпт собирать на бэкенде.

3. **Референс-изображения в одном запросе** — порядок «сначала текст-инструкция, потом картинки» (как multimodal `parts` в их Gemini) для шаблона и лица.

4. **Параметры картинки не только текстом** — aspect ratio и, где API позволяет, разрешение, формат, уровень safety — отдельными полями запроса, а не только словами в промпте.

5. **Маппинг «выбор в продукте → имя модели»** — внутренние id (качество / быстрее / дешевле) → строка модели у провайдера.

6. **Несколько вариантов за один замысел** — у нас уже есть `count`; усилить UX как «сетка из N превью за одно действие».

7. **Два провайдера как опция** — идея основной + запасной путь генерации (у них Replicate vs Gemini); у нас, например, Imagen vs Gemini image — без смены остальной архитектуры.

8. **Явная проверка ответа модели** — убедиться, что в ответе есть изображение; если модель вернула только текст — понятная ошибка для пользователя и логирование.

---

## Не переносить

- API-ключи и вызовы моделей **из браузера** (у нас — сервер и env).
- **CORS proxy** для внешних API.
- История только в **LocalStorage** (у нас — проекты/варианты в БД).
